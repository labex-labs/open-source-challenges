# 添加提交消息模板

如果没有提交消息模板，开发人员可能会倾向于编写模糊或缺乏信息的提交消息，例如“修复了错误”或“更新了代码”。这使得其他人难以理解更改的目的，并可能导致后续的混乱或错误。通过设置提交消息模板，可以鼓励开发人员提供更详细、更有信息的提交消息，从而提高协作效率和生产力。

## 任务

对于这个挑战，我们使用来自 `https://github.com/labex-labs/git-playground` 的仓库。

1. 导航到仓库目录并配置你的 GitHub 身份。
2. 在仓库的当前目录中创建一个名为 `commit-template` 的新文件。
3. 在文本编辑器中打开 `commit-template` 文件，并添加以下行：

```shell
# <类型>: <主题>

# <正文>

# <页脚>

# 这创建了一个包含三个部分的模板：
# "<类型>" 表示提交的类型，例如 "feat" 或 "fix"
# "<主题>" 是对提交内容的简短 #摘要描述
# "<正文>" 是更详细的描述
# "<页脚>" 可以包含其他元数据，例如 #相关的问题编号或其他注释。
```

4. 按下 <kbd>Esc</kbd> 并输入 <kbd>:wq</kbd> 命令，然后按下 <kbd>Enter</kbd> 保存更改并退出 `commit-template` 文件编辑器。
5. 将 `commit-template` 文件添加到暂存区。
6. 将 `commit-template` 文件设置为仓库的提交消息模板。
7. 打开提交消息编辑器，注意提交消息编辑器现在包含你在步骤 3 中创建的提交消息模板。
8. 按下 <kbd>Esc</kbd> 并输入 <kbd>:q</kbd> 命令，然后按下 <kbd>Enter</kbd> 退出提交消息编辑器。

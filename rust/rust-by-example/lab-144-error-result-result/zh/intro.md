# 简介

在本实验中，我们将探索 Rust 中的 `Result` 类型，它提供了一种处理潜在错误的方式，而不像 `Option` 类型那样处理可能不存在的值。`Result` 类型有两种结果：`Ok(T)` 表示成功结果，包含元素 `T`；`Err(E)` 表示错误结果，包含元素 `E`。我们将在代码示例中看到如何使用 `Result`，以及它如何用作 `main` 函数的返回类型来处理错误并提供更具体的错误消息。

> **注意**：如果实验未指定文件名，你可以使用任何你想要的文件名。例如，你可以使用 `main.rs`，并通过 `rustc main.rs &&./main` 进行编译和运行。

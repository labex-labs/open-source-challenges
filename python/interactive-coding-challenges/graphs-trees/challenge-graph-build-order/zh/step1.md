# 图的构建顺序

## 问题

给定一系列项目及其依赖关系，我们需要找到一个有效的构建顺序。构建顺序是一个项目列表，其中每个项目都出现在任何依赖它的项目之前。

## 要求

为了解决这个问题，我们需要考虑以下要求：

- 输入可能包含循环图。
- 我们可以假设已经有了图（Graph）和节点（Node）类。
- 我们可以假设图是连通的。
- 我们可以假设输入是有效的。
- 我们可以假设问题在内存范围内可解。

## 示例

假设我们有以下项目及其依赖关系：

- 项目：a、b、c、d、e、f、g
- 依赖关系：(d, g)、(f, c)、(f, b)、(f, a)、(c, a)、(b, a)、(a, e)、(b, e)

输出应该是：d、f、c、b、g、a、e

请注意，边的方向是向下的，这意味着一个项目依赖于它下面的项目。

```txt
    f     d
   /|\    |
  c | b   g
   \|/|
    a |
    |/
    e
```

如果输入包含循环图，输出应该是 None。

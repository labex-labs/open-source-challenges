# 关闭生成器

关于生成器的一个常见问题是它们的生命周期和垃圾回收。例如，`follow()` 生成器在一个无限的 `while` 循环中永远运行。如果驱动它的迭代循环停止会发生什么？另外，有没有办法提前终止生成器？

修改 `follow()` 函数，使所有代码都包含在一个 `try-except` 块中，如下所示：

```python
def follow(filename):
    try:
        with open(filename,'r') as f:
            f.seek(0,os.SEEK_END)
            while True:
                 line = f.readline()
                 if line == '':
                     time.sleep(0.1)    # 短暂睡眠以避免忙等待
                     continue
                 yield line
    except GeneratorExit:
        print('Following Done')
```

现在，尝试一些实验：

```python
>>> from follow import follow
>>> # 实验：运行中的生成器的垃圾回收
>>> f = follow('stocklog.csv')
>>> next(f)
'"MO",70.29,"6/11/2007","09:30.09",-0.01,70.25,70.30,70.29,365314\n'
>>> del f
Following Done
>>> # 实验：关闭生成器
>>> f = follow('stocklog.csv')
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            f.close()

"VZ",42.91,"6/11/2007","09:34.28",-0.16,42.95,42.91,42.78,210151
"HPQ",45.76,"6/11/2007","09:34.29",0.06,45.80,45.76,45.59,257169
"GM",31.45,"6/11/2007","09:34.31",0.45,31.00,31.50,31.45,582429
...
"IBM",102.86,"6/11/2007","09:34.44",-0.21,102.87,102.86,102.77,147550
Following Done
>>> for line in f:
        print(line, end='')    # 无输出：生成器已完成

>>>
```

在这些实验中，你可以看到当生成器被垃圾回收或通过其 `close()` 方法显式关闭时，会引发 `GeneratorExit` 异常。

另一个探索领域是，如果从 `for` 循环中跳出，是否可以恢复对生成器的迭代。例如，尝试这个：

```python
>>> f = follow('stocklog.csv')
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            break

"CAT",78.36,"6/11/2007","09:37.19",-0.16,78.32,78.36,77.99,237714
"VZ",42.99,"6/11/2007","09:37.20",-0.08,42.95,42.99,42.78,268459
...
"IBM",102.91,"6/11/2007","09:37.31",-0.16,102.87,102.91,102.77,190859
>>> # 恢复迭代
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            break

"AA",39.58,"6/11/2007","09:39.28",-0.08,39.67,39.58,39.31,243159
"HPQ",45.94,"6/11/2007","09:39.29",0.24,45.80,45.94,45.59,408919
...
"IBM",102.95,"6/11/2007","09:39.44",-0.12,102.87,102.95,102.77,225350
>>> del f
Following Done
>>>
```

一般来说，如果需要，你可以跳出正在运行的迭代并稍后恢复它。你只需要确保生成器对象不会以某种方式被强制关闭或垃圾回收。
